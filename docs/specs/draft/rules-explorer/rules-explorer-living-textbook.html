§3 The twelve rules — the program, in full

Rules as data, read as prose

This is the entire liveness brain: twelve stratified Datalog rules, hand-compiled to SQL over bun:sqlite. Each card is generated from the same RULES constant that generates Figure 1 — there is no second copy to drift. Evaluation runs the strata in order, once per 5-second tick, inside a transaction; no recursion crosses a negation.

S1

Ground correlations

4 rules · pure joins over observations · negation forbidden here

Nothing in this stratum judges. It only correlates raw observations into facts-about-workers that later strata may safely negate.

R7worker_deadfired ×2 this tick

A worker is dead when Claude Code itself says so: the job directory is gone, a terminal timestamp is set, or the state is terminal. The daemon never invents a death; it only believes a durable verdict.

worker_dead(T, P) :-
    obs_signal(T, P, _, Job, _),
    ( obs_job(Job, exists: 0)
    ; obs_job(Job, first_terminal_at: FT), FT != null
    ; obs_job(Job, state: S), S in ["stopped","failed","done","blocked"] ).

S4

Escalation ladder

3 rules · negation over intents · the diagnostician sits here

The daemon detects; the agent interprets; the human is a conclusion of last resort. Cooldowns and attempt budgets are facts in the intent table, so storms and retry loops are unrepresentable rather than discouraged.

R10wake_diagnosticianfired ×1 → intent i1

Wake the diagnostician for states the daemon cannot resolve deterministically — never kill an alive worker on suspicion. CTL-792's Layer-4 bounded stall-diagnostician, expressed as a rule.

wake_diagnostician(T, P, "never-started") :-
    wedged_never_started(T, P),

wake_diagnostician(T, P, "stalled-alive") :-
