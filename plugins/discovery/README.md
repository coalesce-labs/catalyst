# Catalyst Discovery Plugin

User research, metrics, experimentation, and prototyping skills — the discovery and research companion to [catalyst-pm](../pm/README.md).

> **Companion to [catalyst-pm](../pm/README.md)** which focuses on strategy, PRDs, priorities, and release sequencing. This plugin handles the upstream work: understanding users, defining the right metrics, running experiments, and rapid prototyping.

## Skills (14)

### User research (5)
- `/catalyst-discovery:user-research-synthesis` — Affinity mapping + JTBD synthesis from raw interview transcripts
- `/catalyst-discovery:user-interview` — Batch transcript processing → structured insights report
- `/catalyst-discovery:interview-guide` — JTBD interview guide creation
- `/catalyst-discovery:journey-map` — User journey or customer lifecycle maps (two modes)
- `/catalyst-discovery:competitor-analysis` — Deep competitor analysis + monthly monitoring (two modes)

### Retention / activation / metrics (6)
- `/catalyst-discovery:retention-analysis` — Cohort retention analysis and intervention recommendations
- `/catalyst-discovery:activation-analysis` — Setup → Aha → Habit funnel diagnosis
- `/catalyst-discovery:metrics-framework` — Leading vs lagging indicator hierarchy
- `/catalyst-discovery:feature-metrics` — STEDII metric selection for feature launches
- `/catalyst-discovery:experiment-metrics` — STEDII metric selection for A/B tests
- `/catalyst-discovery:experiment-decision` — Reversibility/risk tree: A/B test vs ship vs ship+monitor

### Prototyping (3)
- `/catalyst-discovery:prototype` — AI-assisted prototype prompt generator (Lovable / v0 / Bolt / Artifacts)
- `/catalyst-discovery:prototype-feedback` — Built-prototype review with PRD alignment + usability scoring
- `/catalyst-discovery:napkin-sketch` — ASCII wireframes or browser capture

## Notes

`feature-metrics` and `experiment-metrics` share the STEDII framework and are candidates for future consolidation into a single `stedii` skill with a mode argument. Tracked as follow-up work.
